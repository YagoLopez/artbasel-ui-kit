These type of cards make up most of the interface and are key components for logically grouping page content and making it scalable.

## Options

### Less than 4 artworks in a collection

If an album has less than four images in its collection, each image block should revert to using the background color "MCH-Grey-200" with a white (MCH-White) 1px stroke.

<Canvas>
  <div style={{ width: '326px' }}>
    <CollectionsAlbumCard {...lessFourImagesExample} />
  </div>
</Canvas>

### Not shared or followed

The number of items will always be present. If the collection has no items, it would simply display as “0 items.”

<Canvas>
  <div style={{ width: '326px' }}>
    <CollectionsAlbumCard {...albumExample} numberOfFollowers={0} />
  </div>
</Canvas>

### Shared and followed from the creator’s viewpoint

If the collection has been shared and users followed the collection (from a link given to them), its creator will see the number of followers on the Album cover for that collection.

<Canvas>
  <div style={{ width: '326px' }}>
    <CollectionsAlbumCard {...albumExample} viewpoint='creator' />
  </div>
</Canvas>

### Shared and followed from the follower’s viewpoint

The follower of the collection will only see the author's name of its creator.

<Canvas>
  <div style={{ width: '326px' }}>
    <CollectionsAlbumCard {...albumExample} viewpoint='follower' />
  </div>
</Canvas>

### User has no collections found

If a user hasn't created any collections yet, the CTA text includes the following label underneath it. The label will disappear once they've created a collection.

<Canvas>
  <div style={{ width: '326px' }}>
    <CollectionsAlbumCard variant="new" noCollectionText="Get the art you love organised in one place. Share it to get support or inspire others."/>
  </div>
</Canvas>
