# Typography

Typography is our system of custom fonts which helps create clear hierarchies, guide users through the experience and organise information.

## Typography specifications

All typograhy specifications can be accessed from this <a href="https://www.figma.com/file/TENnOP4K38G6r3vDL3emCP/AB---Style-Guide?node-id=676%3A24536" target="_blank">link</a>.

## Fluid Type

We use these Fluid styles for headings depending on the breakpoints. The font for headings is <b>Art Basel Headline</b> and is used to communicate context and important information. It is used in multiple scenarios but never within body text.

### Headings

Headings are accessible through the <code>h1</code>, <code>h2</code>, <code>h3</code>, <code>h4</code> and <code>h1</code> HTML tags.

### Links

HTML Tag: <code>a</code> inside a <code>p</code>

<Canvas>
  <p>
    More information can be found in <a href="#">this link</a>.
  </p>
</Canvas>

Adtionally a class can be used in other contexts to apply the same link style.

Class: <code>.text-link</code>

<Canvas>
  <p>
    This <span className="text-link">link style</span> was applied using the
    class text-link.
  </p>
</Canvas>

An element wrapped by an <code>a</code> tag will not receive the style from a text link (underline on hover), but is still a link. For example, a header tag:

<Canvas>
  <a href="#">
    <h4>This is an h4 header with a link</h4>
  </a>
</Canvas>

But, if the link styling (underline on hover) is desired, the class <code>.text-link</code> can be applied to the <code>h4</code> tag:

<Canvas>
  <a href="#">
    <h4 className="text-link">
      This is an h4 header with a link and hover underline
    </h4>
  </a>
</Canvas>
